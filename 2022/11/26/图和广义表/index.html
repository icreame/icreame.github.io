图和广义表

Created2022-11-26Updated2022-12-03

图的定义和术语

  • 通常记作G=（V,E）
  • 有向图 无向图
    • 在实际应用中，图的弧或边往往与具有一定意义的数相关，称这些数为“权（weight）”
    • 称带权的有向图和无向图为有向网和无向网
  • 完全图
    • e=n（n-1）/2，无向完全图
    • e=n（n-1），有向完全图
  • 稀疏图 稠密图
    • e<nlogn,称为稀疏图，反之称为稠密图
  • 子图
  • 度（TD）：顶点入度和出度之和
    • 入度：ID
    • 出度：OD
    • 图中所有顶点的度数之和为边数之和的二倍
  • 路径和回路
    • 简单路径：经过的顶点都不相同
  • 连通图和连通分量
    • 连通图：若无向图中任意两个顶点之间都存在一条无向路径，则为连通图
    • 强连通图：有向图中任意两个顶点之间都存在一条有向路径
    • 非连通图中各个连通子图称为该图的连通分量

图的存储结构

图的数组（邻接矩阵）存储表示

  • 对于无向图的存储，怎样可以省一半空间？
    用一个长度为n(n+1)/2个一维数组A存储元素，Aij在数组里的下标为(i*(i+1)/2+j)

其他类型邻接矩阵的代码可参考数据结构——图的邻接矩阵法实现代码

图的邻接表存储表示

  • 邻接表是图的一种链式存储表示方法，类似于树的孩子链表
  • 在有向图的邻接表中，从同一顶点出发的弧链接在同一链表中，邻接表中结点的数量恰为图中弧的数目
  • 需要n个头指针，2e个结点(每个结点至少两个域)

图的遍历

DFS(深度优先遍历)

  • 若有n个顶点，e条边，时间复杂度为：
    • 用邻接矩阵存储图：O(n^2)
    • 用邻接表存储图：O（n+e）

BFS（广度优先遍历）

  • 若有n个顶点，e条边，时间复杂度为：
    • 用邻接矩阵存储图：O(n^2)
    • 用邻接表存储图：O（n+e）
  • 图不连通怎么办？
    • 每调用一次DFS(v),就把v所在的连通分量遍历了一遍。BFS也是一样。
 //解决方法
 void ListComponents(Graph G){
     for(each v in G){
        if(!visited[v]){
            DFS(V);    //或者BFS(V)
        }
    }
 }

求迷宫的最短路径问题

连通网的最小生成树

一个连通图的生成树是一个极小的连通子图，它含有图中全部的顶点，但只有足以构成一棵树（不存在回路）的n-1条边。将生成树的各边的权值相加，就是代价，我们将连通网的最小代价生成树称为最小生成树

Kruskal（克鲁斯卡尔）算法

基本思路：按照权值从小到大的顺序选择 n-1 条边，并保证这 n-1 条边不构成回路
具体做法：首先构造一个只含 n 个顶点的森林，然后依权值从小到大从连通网中选择边加入到森林中，并使森林中不产生回路，直至森林变成一棵树为止

  • 主要对边进行操作，其时间复杂度为O（e），适用于稀疏图

Prim（普利姆）算法

基本思想：首先选取图中任意一个顶点v作为生成树的根，之后继续向生成树中添加顶点w，则在顶点w和v之间必须有边，且该边上的权值应在所有和v邻接的边中属于最小的。

  • 主要对顶点进行操作，其时间复杂度为O（n^2），适用于稠密图
