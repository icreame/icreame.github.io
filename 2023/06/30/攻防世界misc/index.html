pyc文件介绍

  • pyc文件是一种二进制文件，是由py文件经过编译后，生成的文件，是一种byte code（python字节码）
  • 它包含了源代码的编译结果和相关的元数据信息
  • py文件变成pyc文件后，加载的速度也会有所提高

pyc文件出现的原因

py文件可以直接看到源码，如果是商业软件的话，源码不适合直接泄露出去的，如果将源代码转换为pyc后，再发布出去就没有问题了。

pyc文件布局

反编译工具uncompyle6

安装

pip install uncompyle6

使用

uncompyle6 -o test.py test pyc

心仪的公司

webshell介绍

https://zhuanlan.zhihu.com/p/137259357

  • webshell是黑客经常使用的一种恶意脚本，其目的就是获得服务器的执行操作权限，比如执行系统命令、窃取用户数据、删除web界面、修改专业等。
  • 简单地说，webshell就是一个asp或php木马后门，黑客在入侵了一个网站后，常常在将这些asp或php木马后门文件放置在网站服务器的web目录中，与正常的网页文件混在一起。
  • 然后黑客就可以用web的方式，通过asp或php木马后门控制网站服务器，包括上传下载文件、查看数据库、执行任意程序命令等。

什么是木马

  • 指一些程序设计人员在其可从网络上下载的应用程序或游戏中，包含了可以控制用户的计算机系统的程序，可能造成用户的系统破坏甚至瘫痪

什么是后门

  • 一台计算机上有65536个端口，那么如果把计算机看作是一间屋子，那么这65535个端口就可以看作是计算机为了与外界连接所开的65535扇门。
  • 每个门的背后都是一个服务。有的门是主人特地打开音节客人的，有的门是主任为了出去访客而开设的，所以剩下的其它门应该都是关闭的，但偏偏由于各种原因，很多门都是开启的。
  • 于是就有人进入，主人的隐私被刺探，生活被打扰，甚至屋里的东西也被搞得一片狼藉。
  • 这扇被悄然打开的门就是“后门”

webshell特点

  • 极强的隐蔽性
    • webshell最大的特点就是可以穿过防火墙，由于被控制的服务器或远程主机交换的数据都是通过80端口传递的，因此不会被防火墙拦截。
    • 使用webshell一般不会在系统日志中留下记录，只会在网站的web日志中留下一些数据提交记录，没有经验的管理员是很看出入侵痕迹的。
  • 权限提升
    • 在服务器没有配置错误的情况下，webshell将在web服务器的用户权限下运行，该用户权限是有限的。
    • 通过使用webshell，攻击者可以尝试通过利用系统上的本地漏洞来执行权限提升，常见的有查找敏感配置文件、通过内核漏洞提权、利用低权限用户目录下可被Root权限用户调用的脚本提权、任务计划等。
  • 持续远程访问
    • 入侵者可以利用webshell从而达到长期控制网站服务器的目的。
    • 若攻击者自行修复了漏洞，以确保没有其他人会利用漏洞，攻击者可以低调地随时哦内阁制服务器。
    • 一些流行的webshell使用密码验证和其他的技术来确保只有上传webshell的攻击者才能访问它。

获取webshell的常见方法

  • 直接上传获得webshell
    • 因过滤上传文件不严，导致用户可以直接上传webshell到网站任意可写目录中，从而拿到网站的管理员控制权限
  • 添加修改上传类型
    • 现在很多脚本程序上传模块不是只允许上传合法文件类型，大多数的系统是允许添加上传类型
  • 利用后台管理功能写入webshell
    • 进入后台后还可以通过修改相关的文件来写入webshell
  • 利用后台数据库备份及恢复获得
    • 主要是利用后台对access数据库的“备份数据库”或“回复数据库”功能，“备份的数据库路径”等变量没有过滤导致可以把任意文件后缀改为asp，从而得到webshell
  • php+mysql系统
    • 后台需要有mysql数据查询功能，入侵者就可以利用它执行SELECT…in TO OUTFILE查询输出php文件
    • 因为所有的数据都是存放在mysql里面的，所以我们可以通过正常的手段把我们的webshell代码插入mysql，再利用SELECT…in TO OUTFILE语句导出shell

webshell网站后门的清除方法

  • 后门的文件可以直接清楚，找到后门文件，直接删除即可
    • 文件名为index.asp、index.php，这类为自动生成SEO类型文件，可以直接删除，如要彻底删除后门，需找生成此文件的源文件
    • 文件内容只有一行，或很少量的代码，此类被称为“一句话后门”
    • 文件内容中存在password或UserPass关键字
    • 另外一些在上传组件目录或上传目录的文件可以这季节删除。如eWebEditor、Editor、FCKeditor、webeditor、UploadFiles、uploads等。
  • 不能直接删除后门文件，只能删除文件内容中的木马代码
    • 网站自身文件被插入恶意代码
    • 网站配置文件
    • 清除方法：首先备份此文件以备改错可恢复，查找到后门代码的位置，一般通过查找“eval、execute、request、ExecuteGlobal”关键字进行定位。把确定为后门的代码删除后保存文件。访问网站是否报错，以确认有没有改错。
